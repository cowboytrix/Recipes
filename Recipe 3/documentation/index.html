In order to successfully open and customize the product you need xCode 7.2:

2. Start customizing - top

When you are ready with the installation of xCode 7.2, just extract the content of the .zip file that you get from CodeCanyon (e.g. dmbteam_ios_joke_app_v1.1.zip) and double click on the Recipe.xcodeproj file. This way you will open the product with xCode.

II. How to Customize "iOS Recipe App"

1. Recipes (Adding Categories and Recipes) - top

First of all you should decide whether you will use a local .xml file (within Supporting Files folder) for your recipes or you want to use external .xml file (web hosted on a web server).

By default the app uses local .xml file and you just need to add your recipes to recipes_settings.xml file within the Supporting Files directory.

1.1. External .xml file (Web Hosted): If you want to use web hosted .xml file (uploaded on a web server), you have to set up the project to do so. For that purpose open (double click) the Constants.swift file and look for the RemoteXMLUrl constant. You should add, as vallue, the path (web url) to your .xml file, like in the example below:

As you can see now it is set to use our test .xml file. So you have to replace the URL http://dmb-team.com/apps/recipe/recipes_settings.xml with yours.

1.2. Local .xml file (within Supporting Files folder): If you want to use a local .xml file you should leave the RemoteXMLUrl constant empty (with no value), like this:

1.3. How to Edit the .XML file (applies for both local and external):

If you are using the local .xml file you can edit it directly within xCode. If you want to use external .xml file, just copy the local one and paste it on your desktop. After that open it via text editor program (Notepad++ or similar) and apply the needed changes. When you are ready you should upload that .xml file to your web server. Please note that you should point the path to the xml file within the constant described above.

To add Recipes, do the following:

  1. Replace the contents of recipes_settings.xml (it is located in the Supporting Files folder) with your own.


    2. Elements description:

    Section <MainSettings>
    → Sub Element <LogoActive' - name of the image to be displayed as active logo in the left menu
    → Sub Element <LogoInactive' - name of the image to be displayed as active logo in the left menu

    Section <Category> - it has two attributes
    - attribute 'name' - name of the Category
    - attribute 'icon' - icon of the Category

    → Sub-Section <RecipeInfo> - represents a single recipe and the info about it. It has ID attribute, which MUST be unique. So each of the recipes you are about to add, MUST HAVE unique IDs.

    -- → Element <RecipeTitle> - put the name of the meal here
    -- → Sub-2x-Section <RecipePictureList > - represent the pictures for the recipe.
    -- -- → Element <RecipePicture> - each picture is stored within such element. If you want to have several images, you have to create several RecipePicture elements within RecipePictureList. Make sure all of your images to be .png and skip the .png file extension when you are setting the value within this element. For example if your image file is my-image-file.png, you have to put here only my-image-file. Of course that applies only for the INTERNAL XML file. When you are about to use EXTERNAL. There you have to place the URL of the image.

    -- → Sub-2x-Section <RecipeSummary> - summary of the recipe; contains the following:
    -- -- → Element <RecipeSummaryOrigin> - origin of the recipe
    -- -- → Element <RecipeSummaryPreparationTime> - preparation time of the meal
    -- -- → Element <RecipeSummaryCookingTime> - cooking time of the meal
    -- -- → Element <RecipeSummaryPortions> - portions of the meal
    -- -- → Element <RecipeSummaryCalories> - calories of the meal
    -- -- → Element <RecipeSummaryDescription> - description of the meal

    -- → Sub-2x-Section <RecipeIngredientsList> - list of recipe ingredients
    -- -- → Sub-3x-Section <RecipeIngredient> - represents a recipe ingredient. Add such item for each of the ingredients.
    -- -- -- → Element <RecipeIngredientsName> - name of the ingredient
    -- -- -- → Element <RecipeIngredientsQuantity> - quantity of the ingredient

    -- → Sub-2x-Section <RecipeStepsList> - list of recipe steps
    -- -- → Sub-3x-Section <RecipeStep> - represents a step to make the recipe. Add such item for each of the steps.
    -- -- -- → Element <RecipeStepName> - name of the recipe step
    -- -- -- → Element <RecipeStepDescription> - description of the recipe step
    -- -- -- → Element <RecipeStepImage> - you can add an image to each step. The value of element should be the name of the image file located in the assets. If you are using a external (web hosted) XML file, you can place as value the URL of the image.
    -- -- -- → Element <RecipeStepVideo> - you can add a video to each step. The value of element should be the youtube URL. Example: https://www.youtube.com/watch?v=Rwjk6zmjP70

Please note that you can have either an image or a video integrated into a step - you can't have both of them.

You can always use our test .xml file as template. It contains 7 categories: Main Courses, Appetizers, Soups, Vegetarion, Deserts, Time-Savers and Other. Plus, we have added a lot of comments within the .xml file to be easier for you to edit it.

2. Splash Screen - top

To customize the Splash Screen do the following:

  1. From the left panel of xCode click on Images.xcassets, then find the LaunchImage item and replace the four images that you see on the right. Please make sure to replace the images with the exact same size. To see the size of each image just right click on each of them and select Show in Finder. There you will see the different sizes that you should provide. It's good idea to preserve the image file names too.

    Finder view: As you can see you have 4 different images for the different iPhone's resolutions.
